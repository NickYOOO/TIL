2. 1주차 주요 개념 키워드 적어보기

-Sourcetree: git 을 편하게 사용하기 위한 도구이다.

-Github: git 프로젝트를 원격으로 올릴 수 있는 저장소 기능을 제공, 커뮤니티 기능도 제공하는 서비스다.

-Git : 버전 관리를 위한 도구, 협업 관리를 위한 도구

-Git 버전관리 : 가장 많이 쓰이는 버전관리 도구로 commit 을 사용해 관리한다.(버전관리:프로젝트 상태가 
              변경되는 정보관리) 

-Git 초기화 : git initialize, 내 로컬컴퓨터의 프로젝트를 Git 이 관리하는 프로젝트로 만듬

-commit : 현재 프로젝트 상태를 스냅샷 찍고 저장!
          내가 수정한 소스를 local 저장소(Local Repositary)에 프로젝트 변경 내용 반영
          누가(author), 언제 커밋했는지 정보공유

-commit msg : 작업내용이 어떤 것이 있는지 내용전달을 할 수 있는 메세지(내용전달이 잘 될수 있게,간결,정확 작성)

-Staging(add) : commit에 반영할지 말지 파일 단위로 선택 가능. Commit 에 반영할 파일을 선택하는 것.

-Commit history : 커밋 내역.

-Repo : repository 리포지토리의 약자, Git으로 관리되는 프로젝트를 Git 에서는 Repo 라고한다.

-Local repository(로컬 repo) : 내 컴에 저장되어 있는 로컬리포

-Remote repository(원격 repo) : Github 처럼 다른 곳에서 접속할 수 있는 공간에 저장되어 있는 원격리포

-Push : 내 local 저장소의 내용을 원격 저장소에 반영 = 로컬 repo 의 commit 들을 원격 repo 에 반영하기

-Pull : 원격 저장소의 내용을 local 저장소로 가져옴 = 원격 repo 의 commit 들을 로컬 repo 로 땡겨오기.
        로컬 repo 에 없는 즉,새로운 commit 내역을 모두 로컬 repo 에 한 번에 반영.

-Clone : 원격 repo 를 내 컴퓨터에 가져와서 초기 repo 세팅하는 것을 clone(복제하기)
       = 로컬 리포에는 없고 원격 리포에 있는것을 다운로드 하는 것을  클론 한다 라고 한다

-Tracking : 로컬 repo 가 원격 repo 를 연결하는 것을 추적(Tracking, 트랙킹/branch tracking)이라 함
          = 로컬, 원격 리포 둘다 있어서 연결한때 로컬 리포가 원격 리포를 트래킹한다 라고 한다.
